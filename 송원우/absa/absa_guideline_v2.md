## 0. 본 문서 사용 방법

라벨러는 처음부터 끝까지 1회 정독한 뒤, 라벨링 중 모호 케이스 발생 시 다음 순서로 참조한다.

```
[모호 케이스 발생]
  ↓
① 6장 모호 표현 매핑 사전 검색  →  매칭되면 즉시 적용
  ↓ (매칭 없음)
② 3장 속성별 트리거 키워드 확인  →  매칭되면 적용
  ↓ (매칭 없음)
③ 4장 전역 룰 적용 (편하다, 부정 톤 등)
  ↓ (해소 안 됨)
④ X(미언급)로 처리하고 메모 컬럼에 사유 기록
```

## 1. 라벨 입력 규칙 (3-class)

### 1.1 클래스 정의

| 코드 | 의미 | 정의 |
|---|---|---|
| `P` | Positive | 해당 속성에 대한 **명시적 긍정** 언급 |
| `N` | Negative | 해당 속성에 대한 **명시적 부정** 언급 |
| `X` | Not Mentioned | 해당 속성을 **언급하지 않음** (기본값) |

### 1.2 v1 → v2 주요 변경

| 변경 항목 | 이유 |
|---|---|
| **U(Neutral) 클래스 폐지** | v1에서 사용 빈도 1~22건, P↔U·N↔U 혼동이 κ를 끌어내림 |
| 양가 감정 처리 → 우세 톤 | "예쁜데 작아요" → 우세 톤이 P면 P, 우세가 N이면 N |
| 사실 진술 → X | "두께는 보통", "검정색 샀어요" 등 → 미언급 처리 |
| 브랜드/헤리티지 정의 협소화 | 명시 트리거가 있을 때만 P (3장 5절) |
| "편하다" 귀속 규칙 명시화 | 단어 단독은 X (4장 1절) |
| 부정 톤 우세 룰 도입 | rating ≤ 2 + 강한 부정어 시 약한 P 시그널 무시 (4장 2절) |

### 1.3 입력 형식 표준

```
대문자 1글자만 입력 : P / N / X
공백·소문자·기타입력 → 자동 X 변환
한 리뷰가 한 속성에서 P와 N 모두 가질 수 없음 (3-class 단일 라벨)
```

## 2. 작업 의사결정 트리

```
[리뷰를 본 직후]
  ↓
Q1. 이 리뷰가 어떤 제품에 대한 후기인가?
   → 본문에서 언급되는 속성을 1차 파악 (사이즈/원단/기능/색/브랜드/가격)
  ↓
Q2. 각 속성별로 다음 순서로 판단:
   ┌─────────────────────────────────────────┐
   │ a) 해당 속성의 P 트리거 키워드가 있는가?   │
   │    YES → P 후보 (Q3로)                   │
   │    NO  → b 단계                          │
   │ b) 해당 속성의 N 트리거 키워드가 있는가?   │
   │    YES → N 후보 (Q3로)                   │
   │    NO  → c 단계                          │
   │ c) 모호 표현 사전에 등재된 표현인가?       │
   │    YES → 사전 매핑 적용                   │
   │    NO  → X (미언급)                      │
   └─────────────────────────────────────────┘
  ↓
Q3. 부정 톤 우세 룰 적용 (rating ≤ 2 + 강한 부정어)
   → 약한 P 시그널이면 X로 강등
  ↓
Q4. 메모: 모호했다면 한 줄로 사유 기록
```

## 3. 속성별 상세 룰

### 3.1 핏/사이즈 (Fit & Sizing)

**정의**: 사이즈 적합성, 라이즈, 압박감, 라인감, Y존 노출 정도

#### P 트리거 키워드
```
정사이즈, 딱 맞다, 잘 맞다, 핏 좋다, 핏 잘 잡힌다, 핏감 좋다,
라인 예쁘다, 라인 살아난다, Y존 커버, 와이존 커버,
허리 편하다(사이즈), 허리 잘 잡, 라이즈 좋다,
적당한 압박감, 잘 잡아준다, 안 흘러내림, 안 처짐
```

#### N 트리거 키워드
```
사이즈 작/큼, 작아요, 커요, 한치수 작/크게, 정사이즈 X,
조임, 끼임, 답답함(사이즈), Y존 부각, 와이존 부각, 부각됨,
허리 말림, 흘러내림, 처짐, 늘어짐(사이즈), 라인 이상함,
사이즈 후기 잘못
```

#### X 처리 케이스
- "편하다" 단독 (맥락 없음) → X
- "마음에 든다" 단독 → X
- 단순 사이즈 교환 사실만 진술 ("M에서 S로 교환") → X
- "사이즈는 보통" → X (사실 진술)

#### 예시
| 텍스트 | 라벨 | 사유 |
|---|---|---|
| "정사이즈로 딱 맞아요" | P | 정사이즈 + 딱 맞 |
| "와이존이 부각되지 않아 좋아요" | P | Y존 커버 |
| "엉덩이가 처져요" | N | 처짐 |
| "한치수 작게 시키세요" | N | 한치수 |
| "M으로 교환했어요" | X | 사실 진술 |

**정의**: 원단 촉감·두께, 내구성(보풀·변색·늘어짐 등 시간 경과 결함)

#### P 트리거 키워드
```
부드럽다, 보들보들, 매끄럽다, 촉감 좋다, 원단 좋다,
두께 적당, 두께감 좋다, 탄탄하다, 탄력 좋다,
보풀 없다, 변색 없다, 안 늘어남, 오래 입어도
```

#### N 트리거 키워드
```
보풀, 보풀 일어남, 늘어남, 변색, 후줄근, 후줄근하다,
까실, 따끔, 비침, 비쳐, 비치는, 후진 원단, 원단 별로,
얇다(부정), 두껍다(부정), 까칠, 거칠다,
세탁 후 변형, 세탁하면, 빨면 (이상)
```

#### X 처리 케이스
- "옷이 좋다" 단독 → X
- "원단은 보통" → X
- 색상·디자인 언급 ("색이 예뻐요") → X (디자인으로 이동)

#### 예시
| 텍스트 | 라벨 | 사유 |
|---|---|---|
| "원단이 부드럽고 매끄러워요" | P | 부드럽다 + 매끄럽다 |
| "세탁 후 보풀이 일어났어요" | N | 보풀 + 세탁 후 |
| "비쳐서 속옷이 보여요" | N | 비침 |
| "원단이 까실하고 몸에 안 붙어 좋아요" | P | 까실하지만 긍정 맥락 |

### 3.3 기능성 (Functional Performance)

**정의**: 신축성, 흡습속건, 통기성, 활동성, 보온/냉감, **착용 쿠셔닝/착용감** 등 운동 및 일상 기능

#### P 트리거 키워드
```
[운동 기능]
신축성, 신축성 좋다, 늘어남(기능), 잘 늘어난다,
흡습속건, 땀 빨리 마름, 흡수 잘 됨, 땀 안 차다,
통기성, 시원하다, 바람 잘 통한다,
활동성, 운동할 때 편함, 움직이기 편함, 잘 늘어나서 편함,
보온, 따뜻하다, 안 춥다, 냉감, 시원함(기능),
땀 안 보임, 안 비침(기능 맥락)

[착용감·쿠셔닝] ★ v2.1 추가
폭신하다, 폭신폭신, 쿠셔닝 좋다, 쿠션감 좋다, 발바닥 편하다,
착용감 좋다, 착용감 편하다, 신고 편하다, 신었을 때 편하다,
발이 편하다, 발에 편한, 발목 편하다
```

#### N 트리거 키워드
```
[운동 기능]
신축성 부족, 안 늘어남, 빳빳, 뻣뻣(의류 맥락),
땀 안 마름, 땀 차다, 답답하다(통기),
운동할 때 불편, 활동 제한, 움직이기 불편,
춥다, 안 따뜻, 너무 덥다,
기능 떨어짐, 기능 별로, 운동복 같지 않다

[착용감] ★ v2.1 추가
뻣뻣하다(신발 갑피), 딱딱하다(신발), 착용감 불편, 발등 아프다,
발이 불편하다, 신었을 때 불편, 쫀쫀하지 않다(착용감 맥락)
```

#### ★ v2.1 착용감 귀속 규칙표 (제품 유형별)

| 표현 | 의류 | 신발·양말 |
|---|---|---|
| "폭신하다/폭신폭신" | 기능성 P (쿠션·패딩) | 기능성 P (쿠셔닝) |
| "착용감 좋다" | 핏/사이즈 P | **기능성 P** |
| "착용감 불편하다" | 핏/사이즈 N | **기능성 N** |
| "신고 편하다" | 해당 없음 | **기능성 P** |
| "뻣뻣하다" | 기능성 N (의류 신축성) | **기능성 N** (갑피 강성) |
| "편하다" (단독, 맥락 없음) | **X** | **X** |

> 판단 원칙: 신발/양말 제품 리뷰에서 "착용감"은 소재(촉감)가 아닌 기능성(신는 경험)으로 귀속한다.

#### X 처리 케이스
- "편하다" 단독 (맥락 없음) → X
- "기능 추가 요청" ("패드 있으면 좋겠다") → X (기능 부족 ≠ 추가 옵션 요청)
- "운동복으로 산 거" 단순 정보 → X
- "세탁·건조기 돌려도 변형 없다" → **소재/내구성 P** (기능성 X)
- "예쁘고 폭신폭신" 중 "예쁘고" → 디자인 P, "폭신폭신" → 기능성 P (다중 속성)

#### 예시 (v2.1 확장 + 캘리브레이션 합의 반영)
| 텍스트 | 라벨 | 사유 |
|---|---|---|
| "신축성이 좋아 운동할 때 편해요" | P | 신축성 + 운동 편함 |
| "땀이 빨리 말라서 좋아요" | P | 흡습속건 |
| "너무너무 편해요" (단독) | X | "편하다" 단독 → 4장 1절 적용 |
| "활동하기 편함" | P | 활동성 |
| "운동복인데 빳빳해서 불편" | N | 빳빳 + 불편 |
| "발바닥도 폭신하고" | **P** | 착용감 쿠셔닝 → 기능성 P |
| "착용감도 좋아요" (신발 리뷰) | **P** | 착용감 = 기능성 P (신발) |
| "발등에 닿는 부분이 뻣뻣해서" | **N** | 뻣뻣 = 기능성 N (신발 갑피) |
| "편하게 잘 신게 돼요" | **P** | "신고 편하다" 패턴 = 기능성 P |
| "쫀쫀하지 않은 느낌" (양말) | **N** | 착용감 불만 = 기능성 N |
| "세탁해도 변형 없고" | X | 소재/내구성으로 이동, 기능성 X |
| **"폭신하고 좋은데 고무줄이 발등을 압박"** | **P** | ★캘리브레이션 합의: 쿠셔닝 P 신호 → 기능성 P. 고무줄 압박은 핏/사이즈 N으로 분리 |
| **"쿠션감이 매우 좋아요"** | **P** | ★캘리브레이션 합의: 쿠션감 = 기능성 P (v2에서 X였으나 v2.1 규칙 적용) |
| **"운동화 끈 조이는 게 뻣뻣해서 불편"** | **X** | ★캘리브레이션 합의: 끈 소재 강성 = 소재/내구성 N, 기능성 X |

### 3.4 디자인 (Design & Aesthetics)

**정의**: 색상, 패턴, 실루엣, 데일리 활용도, 시각적 매력

#### P 트리거 키워드
```
예쁘다, 이쁘다, 이뻐요, 예뻐요, 디자인 좋다, 디자인 마음,
색깔 예쁘다, 색감 예쁘다, 컬러 예쁘다, 색이 좋다,
패턴 예쁘다, 실루엣 예쁘다, 라인 예쁘다(시각),
데일리, 일상에서, 데일리 룩, 출근 룩, 코디 잘 됨,
어울린다, 여성스럽다, 세련됨, 고급스럽다(디자인 맥락)
```

#### N 트리거 키워드
```
안 예쁘다, 디자인 별로, 색감 별로, 색 별로, 컬러 이상,
패턴 별로, 실루엣 이상, 사진과 다름,
촌스럽다, 안 어울림, 너무 화려, 너무 밋밋
```

#### X 처리 케이스
- 색상 단순 언급 ("검정색 샀어요") → X
- 디자인 무관 만족 → X

#### 예시
| 텍스트 | 라벨 | 사유 |
|---|---|---|
| "색깔이 너무 예뻐요" | P | 색 예쁘다 |
| "데일리로 입기 좋아요" | P | 데일리 |
| "사진이랑 색이 달라요" | N | 사진과 다름 |
| "검정색 주문했어요" | X | 사실 진술 |
| "양면 착용 가능해서 맘에 들어요" | P | 디자인 만족 (브랜드 X) |

### 3.5 브랜드/헤리티지 (Brand & Heritage) — **v2.1 협소화 강화**

**정의**: **명시적인** 재구매 의도, 추천, 브랜드 충성도, 정체성 표현
> ⚠️ 주의: 단순 만족·칭찬은 디자인 또는 핏으로 분류, 브랜드는 X

#### ★ v2.1 핵심 원칙 (반드시 숙지)

```
브랜드/헤리티지 P = "이 브랜드를 또 사겠다 / 추천한다 / 팬이다"
                    라는 브랜드 지향 표현이 명시적으로 있을 때만 P

브랜드/헤리티지 N = "이 브랜드에 실망했다 / 다시는 안 산다"
                    라는 브랜드 신뢰 상실 표현이 있을 때만 N

그 외 모든 제품 칭찬·불만 = 해당 속성(핏/소재/기능/디자인)으로 귀속, 브랜드 X
```

#### P 트리거 키워드 (반드시 명시어 있어야 P)

**재구매 의도** (가장 자주 등장)
```
또 살, 또 살게요, 또 사고 싶, 다음에도 살, 다음에도 살게요,
재구매, 재주문, 재구매 의사, 한 번 더, 또 하나 살,
{브랜드명} 또 살, 같은 거 또
```

**추천 표현**
```
추천, 추천드립, 추천드려요, 추천합니다, 강추,
{브랜드명} 추천, 자주 사
```

**충성도·헤리티지**
```
믿고 산다, 믿고 사는, 팬이다, 팬, 단골, 단골 브랜드,
여기만 산다, 여기 거만, {브랜드}답다, {브랜드}스럽다,
{브랜드} 감성, {브랜드} 느낌, 안다르답다, 안다르 같다,
젝시믹스답다, 휠라스럽다, 룰루레몬 느낌, 올드머니룩
```


### 3.6 가격/가치 (Price & Value)

**정의**: 가격 적합성, 가성비, 프로모션·할인 인식

#### P 트리거 키워드
```
가성비, 가성비 좋다, 가성비 갑, 이 가격에, 이 가격이면,
저렴하다, 합리적, 가격 착함, 가격 만족, 할인 좋다, 세일
```

#### N 트리거 키워드
```
비싸다, 너무 비쌈, 가격 대비 별로, 가격이 부담,
이 가격에 이걸, 가성비 X, 가격 거품
```

#### X 처리 케이스
- 가격 언급 없음 → X (대부분 케이스)
- 단순 결제·할인 정보 ("3만원에 샀어요") → X

## 4. 전역 룰 (Cross-cutting Rules)

### 4.1 "편하다" 귀속 규칙 (★ Hard Rule)

`편하다`는 한국어에서 핏·기능·소재·일반 만족 등 다의적이다. 다음 규칙으로 자의성을 차단한다.

| 맥락 키워드 | 귀속 속성 |
|---|---|
| "운동할 때 편함", "활동하기 편함", "움직이기 편함" | **기능성** P |
| "신고 편함", "착용감 편함" (신발·양말 제품) | **기능성** P ★v2.1 |
| "착용감 편함", "허리 편함", "사이즈 편함" (의류) | **핏/사이즈** P |
| "원단이 편함", "감촉이 편함" | **소재/내구성** P |
| "그냥 편해요", "너무 편해요" (단독, 맥락 없음) | **모두 X** |

**판단 원칙**: "편하다" 앞뒤 ±5어절에 명시 맥락 단어가 없으면 X로 처리한다.

> ★ v2.1 추가: 신발/양말 제품 리뷰에서 "신고 편해요", "편하게 신게 돼요" 패턴은 착용감(기능성)으로 귀속한다.

### 4.2 부정 톤 우세 룰 (★ Hard Rule)

**조건**:
- `rating ≤ 2` AND
- 본문에 다음 강한 부정어 1개 이상 포함:
  ```
  별로, 후줄근, 실망, 기대 이하, 최악, 환불, 반품, 비추,
  짜증, 후진, 망함, 쓰레기, 짜증나
  ```

**효과**: 약한 P 시그널은 X로 강등
- "추가구매했는데 별로예요" → 브랜드 X (재구매 의도 무효)
- "또 살까 했는데 실망" → 브랜드 N (완전 부정 시그널은 N 유지)

### 4.3 다중 속성 매핑 허용

한 표현이 두 속성에 모두 매핑 가능하면 **두 속성 모두 라벨링**한다.

| 표현 | 매핑 |
|---|---|
| "신축성이 좋아 핏이 잘 잡힌다" | 기능성=P + 핏/사이즈=P |
| "디자인도 예쁘고 가성비도 좋아요" | 디자인=P + 가격/가치=P |
| "재구매했는데 사이즈가 작아요" | 브랜드=P (재구매) + 핏/사이즈=N |

### 4.4 별점 vs 본문 우선순위

| 충돌 | 우선 |
|---|---|
| rating=5 + 본문 부정 | **본문** ("불편한데 별점은 5점")  |
| rating=1 + 본문 긍정 | **본문** (대체로 별점 오타) |
| 단, 부정 톤 우세 룰(4.2)에서는 rating도 조건으로 사용 |

## 7. 작업 절차 및 합의 프로세스

### 7.1 라벨러 작업 시간 가이드

| 작업 | 1리뷰 평균 시간 | 1,000건 누적 |
|---|---|---|
| 정독 + 라벨링 | 30~45초 | 8~12시간 |
| 모호 케이스 메모 | +15초 (해당 시) | +1시간 |

### 7.2 합의 프로세스

```
[Phase 1] 가이드라인 v2 정독 (모든 라벨러)
[Phase 2] 캘리브레이션 50건 — 라벨러 2명 독립 라벨링
[Phase 3] κ 측정 → κ ≥ 0.65 도달 확인
   ├ 미달 시: 불일치 케이스 토론 → 가이드라인 v2.1 보강
   └ 도달 시: Phase 4 진행
[Phase 4] 본 라벨링 1,000건 (분담 또는 전수 독립)
[Phase 5] 100건 교차 검증 → κ 최종 측정
[Phase 6] 불일치 케이스 → 시니어 중재 → 합의 라벨 확정
```

### 7.3 P↔N 정반대 라벨 처리

라벨러 2명이 같은 리뷰에 정반대 라벨(P vs N)을 단 경우:
- **자동 시니어 중재 대상** (다수결 적용 X)
- 시니어가 본문을 직접 검토하여 합의 라벨 결정

### 7.4 P↔X / N↔X 불일치 처리

- 다수결 적용 가능 (라벨러 2명 + 시니어 1명 → 다수)
- 단, 같은 속성에서 P↔X 불일치가 5건 이상이면 **트리거 사전 보강** 필요 신호

## 8. FAQ

**Q1. 리뷰가 너무 짧고 정보가 없으면?**
→ 모든 속성 X. 메모에 "정보 부족" 기록.

**Q2. 다른 제품과 비교하는 표현은?**
→ 비교 대상이 같은 브랜드면 비교 표현 그대로 라벨링. 타 브랜드 비교는 메모 기록 후 X 우선.

**Q3. 이모티콘만 있는 표현 (ㅎㅎ, ㅋㅋ)은?**
→ 이모티콘은 라벨 근거 아님. 본문 텍스트만 기준.

**Q4. 사이즈 정보 본문 + 만족 표현이 동시에 있으면?**
→ "M사이즈 잘 맞아요" → 핏/사이즈 P. "M사이즈 샀어요"만 → X.

**Q5. 별점은 1점인데 본문은 칭찬?**
→ 4장 4절(별점 vs 본문 우선순위) 적용 → 본문 따라가되 메모.

**Q6. "편하다"가 같은 문장 안에 두 번 등장하면?**
→ 각각 맥락 키워드 확인하여 매핑. 다중 속성 가능.
